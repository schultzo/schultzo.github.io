Problem Solving

Pseudocode

For more complex programming tasks Pseudocode can really help to make sure what you are trying to achieve is clear in your mind and you understand the problem you are trying solve. It is written in plain english each step you are trying to achieve. In the video I watched they mentioned that as you become more familiar with the language you are writing the code in you can start to complete the Pseudocode in the coding language

Code Readability

I watched a useful video called "Writing Readable Code" which described some helpful tips to make your code more readable when future you or someone else on the same project needs to figure out what's going on.

  • Keep the line length 45-90 characters long - be mindful that different people have different set ups and while it might be easy to read on your large work screen, someone else may use a much smaller screen and find it difficult to follow along.
  • Try and write code that doesn't require a comment - by having the goal of your code not requiring a coment you strive to make it as naturally readable as possible
  • The following words are generic words that shouldnt be used in variable/function names as they are ambiguous - check, validate, make, get/set, process, start
  • Try and align with the way that existing code is already written - Generally you will be working on projects with other people, and with an existing code base. By seeking to understand the way the code is currently written you can help to ensure consistency going forward.

Testing

Test Driven Development

With Test Driven Development (TDD) you write a test for your code before you even write it. In the video "Problem-Solving for Developers" the described the process as: Write a test that describes the expected output of the test. Write a failing test, implement code that pasts the test then optimise your code.

  1. Write a test that checks the expected output of the test vs an actual output
  2. Write a failing test - this allows you think differently about the problem and ensure you understand it correctly
  3. Implement the code to pass the test
  4. Optimise your code - make it more efficent, increase code readability
