What is javascript?

Javascript is ACTION, it allows a website to be more interactive and dynamically change. It can change the CSS and HTML allowing what is display to user a change based on input or certain action on the site. Using the analogy of a car the Javascript is the engine. The HTML provides the stucture and defines the number of seats, how many lights, how many windows whilst the CSS determines how all of those elements are styled on the car e.g. 2 in the front, 3 in back with colour of them being black. CSS my define the windows to be closed. Then when Javascript comes along it provides the action which causes windows to go up, the car to move when accelertor is pushed and other actions which makes a car functional. In a magical world it would also turn the car seats a different colour!

WHAT THE FU nction?!!!

A function is a chunk of javascript code which can be reused by wrapping it in a function. This allows you to just "call" that function instead of rewriting all the code again!

Declaring a function

First thing you need to do is "delcare" the function. This gives it a name that you will use to call it (use it) later on. Inside the curly brackets {} you will write the code you want to run when the function is called. You can also "pass" it variables which allow for the outcome of the function to be different depending on the variable/s passed. 

                
                    function myFunction(myVariable){
                        alert("You have run myFunction!" + myVariable);
                    }

Calling a function

Throughout your application you may want to use that code you wrote previously when you "declared" your function. Because you did this you do not need to write all the "doing" code again, instead you just have to remember what you named it as and write that. With the below example if you had declared it as per above 

                
                    myFunction("Fantastic!");

The above code would display something like this!

Control flow - what's that?!

Control flow is the order in which the code you write is executed. The order in which you write things matters and if written in the wrong way can cause an outcome which you didn't expect or want!

GOOD

In the below example I want to fill up my petrol tank only when it's empty because what a waste of time trying to fill up an already full tank
Below we can see that fillUpPetrol is called and run INSIDE the if statement to check to see if petrolTank is empty and results in the car to only be filled up if the tank is empty. 
                
                    var petrolTank = "empty"; 


                    function fillUpPetrol(){

                        petrolTank = "full";

                        console.log( "Petrol has been filled");
                    }

        
                    
                    if (petrolTank == "empty"){

                        fillUpPetrol();

                    }

BAD

In the below example fillUpPetrol is called and run before the if statement to check if the the petrol tank is empty! Because it is done before we will always be trying to fill up a tank that's already full. 
                
                    var petrolTank = "empty";


                    function fillUpPetrol(){

                        petrolTank = "full";

                        return "Petrol has been filled";
                    }

        
                    fillUpPetrol();

        
                    if (petrolTank == "empty"){

                        alert("Your tank is empty");
                    }

Where is Javascript written

Similar to CSS you can have an external js file which is linked to your HTML or you can have it in the page. If you link your HTML to an external javascript file best practice is to put it at the end of the file. (Just before the closing body tag). The reason for this is because of the way the browser reads your file (from top to bottom). If script tag is in the head then when your browser loads the HTML file it will wait until the JS file is loaded until continuing to process rest of your HTMl.This could cause issues if your javascript does something on load to an element that has not yet been created as broswer has not read that part of the document yet. Also, if your Javascript file is really complex it may affect the load time of the page causing a worse user experience.
