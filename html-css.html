What are the differences between relative, absolute, and fixed positioning?

Overall there are 5 different positioning values for the "Position" property.

  • static
  • relative
  • fixed
  • absolute
  • sticky
In this article I will cover the relative, absolute and fixed values and their effect on how changing to these could effect your webpage.

In the world of web everything is made up of containers which are positioned on a screen which is displayed to the user. Out of the box the way these containers are layed out depends on the order in which you write your code using the default styling for each tag. For a better experience we may want certain elements to display in ways that are the default positioning. This is when we can use the "Position" property in CSS to explicitly state how we want the element to positioned on the screen.

Fixed

Relative

When something has position:relative it will be positioned relative to where it would be positioned if no explicit positioning was applied. The effect on the other elements is that it will leave a space where that element used to be

Absolute

it ABSOLUTELY must be positioned exactly as the developer as stated. If the developer writes: left:200px; that item must be 200px away from the left regardless of how it would be otherwise positioned. The effect on the other elements is though the absolute positioned element was never there.
